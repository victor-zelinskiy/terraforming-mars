**Iteration 5 — fact-based endgame STORYTELLING + premium UI (the facts now drive «Как сложилась партия»).** The fact layer is WIRED into the endgame insight engine (`src/client/components/endgame/insightEngine.ts`): `EndgameExperience.vue` fetches `/api/game/endgame-facts` once on mount + derives `playerCards` from the view, passing both through `buildEndgameModel` → `generateInsights(ctx.facts, ctx.playerCards)` (graceful: undefined facts → the base template analyzers, no error). `InsightCandidate` gained OPTIONAL story metadata — `family` (15 families), `uiVariant` (hero/legendary/major/normal/compact/…), `storyCluster`, `scores` (`impact`/`rarity`/`drama`/`confidence`/`relevance`), `related*`, + selector-set `rankSection`/`finalScore`. The NEW selector `selectStoryInsights` ranks by `finalScore` (priority + a fact-score bonus SCALED by confidence, so a `partial` fact can't dominate), picks ONE HERO (a decisive verdict / rarity≥0.7 / drama≥0.8), a diverse PRIMARY band (≤1 per cluster, rare ≥0.6 breaks through), a SECONDARY band, and HIDDEN ("show more"). 7 FACT analyzers (`FACT_ANALYZERS`) read `ctx.facts`: economy (engine + **underdog win**), blueAction (most-used + **unused engine**), negativeDrama (most-targeted + **Predators** raid), **Vermin** (city-pressure via `playerCards`), globalParameter, reveal, colony — each thresholded + honest with confidence ("measured value"/units, never fake M€/VP; Vermin = "pressure", no fabricated VP). Tiebreaker/photo-finish/late-comeback are elevated to the HERO. `EndgameOverviewTab.vue` renders the hierarchy (hero card + key-moments grid + compact details + "Show more analysis") with per-family accent tints (`endgame.less` `.eg-fam(...)`). The 9 base template analyzers stay as the fallback layer. Guards: `tests/client/components/endgame/factInsights.spec.ts`; existing `insightEngine`/`endgameModel`/`EndgameOverviewTab` specs unchanged.

**Iteration 6 — DEEP story expansion (12 fact analyzers).** Added on top of the Iteration-5 wave: **runner-up story** (`analyzeRunnerUpStory` — the category the runner-up out-scored the winner in vs. the winner's answer; + "better economy, couldn't convert"), **category structure** (`analyzeCategoryStructure` — TWO-PILLAR win, suppresses the soft `reason.dominant-category`; or SECONDARY strength — no category spam), **standard-project strategy** (a NEW server fact `standardProject` in `endgameFacts.ts` = per-player project count + the parameter steps they drove from `category:'standard-project'` roots — NO invented VP), **unused potential** (`analyzeUnusedPotential` — a big leftover `player.megacredits` pile, "money with nowhere to go"), **notable moments** (`analyzeNotableMoments` — the biggest one-off beats, e.g. a single-generation economy burst). **Vermin 2.0** reads the actual ANIMALS on Vermin (threaded via `ctx.cardResources` from the view) — a played-but-empty Vermin yields NO insight (false-positive FIX); ≥4 animals + city-heavy opponent → rare (never a fabricated VP, only "pressure"). **Predators 2.0** is source-aware + threshold-tiered (6+ = rare raid, 3–5 = normal hunt). **Selector tuning:** PRIMARY sorts by `finalScore` (strongest story first), legacy `GROUP_ORDER` only a tiebreak — fact insights no longer buried behind legacy groups. **Debug:** `buildInsightCandidates(ctx)` returns every candidate (base+fact) with `finalScore` BEFORE selection (the rendered insights also carry `finalScore`/`rankSection`/`family`). Guards: `factInsights.spec.ts`.

**Iteration 7 — DUEL-specific storytelling (`mode === 'duel'`).** Tells a 2-player RIVALRY story, gated to duel (multiplayer untouched). KEY DATA FINDING: `VictoryPointsBreakdown.detailsAwards` rows carry `messageArgs = [place, awardName, funderName]` + `victoryPoint` (and `detailsMilestones` carry `[milestoneName]`), so the award race ("funded by A but won by B", swing > margin) + milestone race (who claimed) are RELIABLY read from `ctx.players[].breakdown` — no fragile text parsing, no new outcome fact (only lightweight `milestoneClaim`/`awardFunding` facts in `endgameFacts.ts` for the generation/timing). Duel analyzers (in `FACT_ANALYZERS`, all gate on `isDuel(ctx)` + set a high `duelRelevance`): `analyzeDuelStyleContrast` (a `duelStyle()` label per player from facts+strongest category → "Two plans: A as Terraformer, B as Card Engine" — the duel HERO, `family:'duelContrast'`), `analyzeAwardRace` (sponsor-lost / swing), `analyzeMilestoneRace` (lockout / contested), `analyzeDuelCategoryCounterplay` (A took X, B answered Y), `analyzeDuelEconomyConversion` ("not richer — more efficient", `suppresses` the generic underdog), `analyzeDuelAlmost` (penalties cost the match / leftover M€ > gap, honest framing). New `InsightScores.duelRelevance` (0..1) folds into `finalScore` (+30 max). **Selector:** the suppression PRE-PASS now runs by `finalScore` order (a strong PRIMARY candidate can suppress a weaker HERO-worthy one — so the duel conversion cleanly replaces the generic underdog). **UI:** `EndgameOverviewTab` renders a rivalry VS row (player A · VS · player B dots) on a `duelContrast` hero + a violet family accent. Honest: no fake VP/M€; "almost claimed" milestone progress is NOT claimed (only confirmed claims + margin relation). Guards: `factInsights.spec.ts` → 27 (style contrast, award sponsor-lost/swing, milestone lockout, counterplay, economy conversion + underdog suppression, almost-penalty, "selector surfaces BOTH players"). (Iteration-8 added the special-card registry.)

**Iteration 8 — Special Card Story Registry (source-aware card attacks).** An EXTENSIBLE registry `src/client/components/endgame/specialCardStories.ts` (`SpecialCardStory = {id, detect(ctx)}` + `SPECIAL_CARD_STORIES` + one `analyzeSpecialCardStories` analyzer in `FACT_ANALYZERS`; TYPE-ONLY engine imports → no module cycle; self-contained param/fact helpers; PURE). Notices "this card DID something notable", never "was played" — every story thresholded + evidence-based. **The genuine gap it fills = SOURCE-AWARE attacks** (which CARD broke whose engine), via a new bridge: `aggregate.aggregateAttacksBySource` (per attacker→sourceCard→victim→scope loss totals + transfer flag, from each victim-loss event's CARD source — reliable for destroys/production-reductions; player-only steals lack a card source → omitted, no guessing) → a `cardAttack` fact in `endgameFacts.ts`. First wave (4): **productionSteal**, **resourceOnCardDisruption** (microbe/floater — animals EXCLUDED, Predators owns them; `rareEvent`), **plantDenial**, **counterStyle** (plant attack on a board-heavy victim — `suppresses` plantDenial). **DELIBERATELY NOT re-added (already covered, avoiding spam — see the audit in `docs/EVENT_STAT_FOUNDATION.md`):** huge single-card VP (`analyzeCards`), penalty tradeoff (`cards.penalties-cost`/`duel.almost.penalty`), economy monster (`analyzeEconomyFacts`), card-flow (`analyzeRevealFacts`), colony (`analyzeColonyFacts`), standard projects (`analyzeStandardProjectStrategy`), blue action (`analyzeBlueActionFacts`), Vermin/Predators (2.0). Honest: "could have scored" (potential, no fake VP); thresholds (production ≥2, card-resource ≥4, plants ≥6) avoid spam. **To add a new card story:** append a `SpecialCardStory` to the registry (a thresholded `detect` reading `ctx.facts`/breakdown) — no engine edit. Guards: `factInsights.spec.ts` → 34 + `factAggregates.spec.ts` (`aggregateAttacksBySource`). **Iteration-9 frontier:** per-source steal attribution; a rules-grounded "prevented greenery"; final-inventory (steel/titanium/cards) bridge; more counter matchups; a fuller attacker/victim-split UI card; a dev candidate-scoring panel (data exposed via `buildInsightCandidates`).

all gate on data + thresholds, confidence-aware, **never invent VP/M€** — compare KNOWN quantities): `analyzeEconomyConversionMismatch` (economy → low card scoring), `analyzeCardFlowToCards` (reveal flow → cards category), `analyzeGlobalParamMismatch` (top terraformer ≠ winner / won without the planet), `analyzeAttackDamagedStrategy` (biggest attack → board-heavy victim's plan), `analyzeProjectCardStarvation` (heavy SP + low card-VP = projects in place of a card engine — grounded in the cards CATEGORY, NOT the rare `reveal` facts; rare-breakthrough keeps rare events out of `hidden`). Guards: `tests/client/components/endgame/gameStoryDna.spec.ts` (23 — detection per shape, composer hero-alignment/penalty/roles/solo, the 5 cross-facts, + a DISTINCTNESS guard: 6 fixtures → ≥5 distinct storyTypes, rare event never hidden, duel names both players, quiet game graceful) + `EndgameOverviewTab.spec` story-headline assertion. i18n: 48 new `ru/endgame.json` keys ("Standard projects" reused from `ui.json`). **To add a new story shape:** add a `StoryDetector` (ordered, most-special-first) + its `headlineKey`; a new cross-fact = a `FACT_ANALYZERS` entry emitting a new `storyCluster`. **Iteration-10 frontier:** a dedicated UI "player arcs" section (DNA already computes `playerArcs`); momentum/turning-point cross-facts; richer `whyRunnerLost`; a production PROFILE in ctx (unlocks production-steal-vs-energy counters); the carried-over per-source steal + final-inventory bridges; a full dev candidate-scoring panel.

**Iteration 15 — Directed STORY layer: key episodes + the unique story of THIS game.** The overview stopped being a packaged pile of facts and became a directed post-game report. **100% client-side** (the timeline's "engine online" beat is INFERRED from VP-by-generation acceleration — passive effects are already journal-logged via `effect-triggered` roots + the `passiveEffect` fact, but carry no per-generation "came online" pin, so VP acceleration is the honest signal; documented in `?egDebug`). **(a) Key-episode engine** `keyEpisodeEngine.ts` (PURE; type-only engine imports) — `buildKeyEpisodes(ctx)` turns the game into ORDERED, role-CLASSIFIED, impact-GRADED episodes from the strategy profiles + the `vpByGeneration` timeline + the facts + the scoring breakdown. `EpisodeRole` = decisive_driver / turning_point / signature_moment / ironic_twist / near_miss / structural_contrast / late_scoring / tempo_shift / engine_online / missed_conversion / flavor_only.
